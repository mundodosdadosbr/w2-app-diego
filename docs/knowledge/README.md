## Como manter

- Todo PR que muda comportamento pedagógico, prompt, regra de negócio ou UX **deve** atualizar o documento correspondente aqui.
- Discrepâncias com o código devem ser resolvidas a favor do código + update do knowledge, ou vice-versa com intenção.
- Em caso de mudança de decisão arquitetural, criar/atualizar ADR e referenciar aqui.
